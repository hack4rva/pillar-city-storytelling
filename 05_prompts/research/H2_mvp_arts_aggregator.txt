Section H — MVP Feasibility
Title: Feasibility of Building a Richmond Arts Event Aggregator in 48 Hours

The technical path to a Richmond arts event aggregator starts with data source selection, and the single most important decision is which source to treat as the spine of the system. Eventbrite is the most API-accessible option: its public search API allows querying events by city, date range, and category without requiring user authentication, and it returns structured JSON including event name, description, start/end time, venue address, and URL. A team can have a working Eventbrite data pull within 2-3 hours of starting. The limitation is coverage: Eventbrite-hosted events skew toward mid-size ticketed events and larger organizations. Many Richmond gallery openings, free neighborhood cultural events, and artist-run space events are not on Eventbrite at all. Teams must research which specific Richmond arts organizations and venues are most active on Eventbrite to assess whether the API spine will produce a compelling enough dataset for demo day.

Facebook Events was historically the richest source of Richmond arts event data — small gallery openings, neighborhood block parties, artist studio events, and informal cultural gatherings that never appear on Eventbrite are routinely posted as Facebook Events. However, Facebook dramatically restricted its Events API access in 2018 following the Cambridge Analytica fallout. Public event discovery through the API now requires app review, business verification, and user-granted permissions that are not achievable in a hackathon context. Teams that plan to "use Facebook Events" as a data source need to understand this restriction clearly before committing the approach. The practical alternative is to manually identify and bookmark 10-15 key Richmond arts Facebook pages and include their events in a seed dataset, which is not automated but is demonstrable. Teams should research which Richmond arts organizations maintain both Facebook Events and a website calendar, allowing manual curation as a bridge.

RSS feeds and iCal/ICS exports are underutilized but genuinely accessible. Several Richmond arts venues and cultural institutions maintain event feeds: the Virginia Museum of Fine Arts (vmfa.museum), the Institute for Contemporary Art at VCU (ica.vcu.edu), the Black History Museum and Cultural Center of Virginia, and mid-size venues like The National and Broadberry post iCal feeds. These feeds are machine-readable and require no API key. The challenge is format heterogeneity — each feed has different field names, date formats, and levels of detail — requiring a normalization layer that takes time to build. A realistic 48-hour approach combines Eventbrite API (for volume and structure) with 3-5 manually identified iCal feeds from anchor Richmond cultural institutions (for legitimacy and coverage depth), seeded into a simple database or even a static JSON file. The demo does not need to be a live cron job — a successful demo can be a manual pull executed the morning of demo day, showing events from that week.

- Inferences (clearly labeled): [Inference] Eventbrite API alone will return enough Richmond arts events for a convincing demo (estimated 15-30 events in any given week), though coverage will be biased toward larger ticketed events. - Unknowns (what cannot be verified publicly): Current Eventbrite API rate limits and whether category codes for "arts" and "cultural" events return Richmond-specific results; whether Richmond's ArtsRVA or similar local arts advocacy organizations maintain a machine-readable event database that could serve as a better spine than Eventbrite.
