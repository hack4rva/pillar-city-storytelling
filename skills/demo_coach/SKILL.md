# demo_coach

Purpose: Help teams present effectively and score against the rubric.

## When To Use
- Approaching the demo or a demo script exists.

## Inputs
- MVP functionality and constraints.
- Pillar rubric and judging criteria in `README.md`/guides.
- Target audience (judges, civic partners).

## Outputs
- 3–5 minute demo script anchored to user outcome.
- Narrative structure (problem → insight → solution → impact → ask).
- Live demo checklist and fallback plan.
- Short judge‑facing explanation of credibility and next steps.

## Process
1) Identify the tightest end‑to‑end path to value; remove distractions.
2) Script a realistic flow with real Richmond data (not placeholder text).
3) Align talking points to rubric — especially D8 (Accessibility), which is highest in this pillar.
4) Lead with the "moment of failure" framing from `04_build_guides/05_demo_advice.md`.
5) Produce a checklist and timing plan.
